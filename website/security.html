In the browser

  • Forced TLS for browser connections
  • All cookies are session level, HTTP only, and have the secure flag, path, and domain values set.
  • HTTP request header fingerprinting and site/user specific page keys.
  • Security related HTTP headers like X-XSS-Protection, X-Content-Type-Options, and Content-Security-Policy.
  • Configurable idle timeout module set that ends a session after a predetermined amount of time.
  • Optional AES encrypted AJAX responses.
  • Optional AES encrypted local storage cache.

On the server

  • TLS/STARTTLS support for IMAP, POP3, and SMTP connections.
  • Oauth2 over IMAP/SMTP supported (currently only Gmail and Outlook support this feature).
  • Session level data is encrypted with a long random string generated on login and stored in a session level cookie.
  • Persistent data stored between logins is encrypted with a key based on your clear text password, which is obviously not stored anywhere.
  • All encryption is done with AES-256-CBC, encrypt-then-MAC, and PBKDF2 key derivation
  • Only white-listed and typed/sanitized input is allowed. Modules must pre-define input in order to have access.
  • Modules can't reference PHP super-globals, and there are no globally scoped variables at all (there are a few static class instances with private data structures that use getter/setter methods).
  • PHP ini settings are tightened up at runtime for extra security, including open basedir and session best practices (at least the ones that can be restricted at runtime).
  • No writable files or directories are used inside the web-server document root, and only 3 files need to be inside the document root to run the program. Module sets can include additional assets, such as the HTML editor for outbound mail.
  • Optional local DB based authentication using a salt and PBKDF2.
  • Development is done with PHP's "E_ALL" and "E_STRICT" error reporting.
  • Easy output escaping inside modules for potentially dangerous content.
  • Module design enforces a separation of input processing and output formatting, and encourages concise methods, which makes the code easier to debug and audit.
  • 100% unit test coverage with PHPUnit for the application framework.
  • HTML formatted E-Mail is filtered through HTMLPurifier with all external resources removed before being rendered.
